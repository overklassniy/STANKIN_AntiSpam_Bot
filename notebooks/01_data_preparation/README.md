# 01. Подготовка данных

Ноутбуки для загрузки, очистки и препроцессинга обучающего и тестового датасетов.

## Ноутбуки

### 01a_data_loading.ipynb

Загрузка и объединение данных из всех источников:

- **Ручная разметка** — `dataset.json` + экспорты чатов (ham)
- **Discord** — `dataset_discord.json` (spam)
- **tg-spam (umputun)** — `spam_samples_umputun.txt` (spam)
- **lols.bot** — `parsed_lols_2024.txt` (spam)
- **Незарегистрированный спам** — из SQLite базы бота

После объединения проверяются конфликты меток. Результат сохраняется в `data/interim/raw_combined.csv`.

### 01b_data_cleaning.ipynb

Очистка объединённого датасета:

1. Удаление пустых и коротких текстов (< 5 символов)
2. Фильтрация текстов без буквенно-цифровых слов
3. Точная дедупликация по `text`
4. Near-дедупликация (нормализованная)
5. Near-дедупликация (MinHash LSH, Jaccard >= 0.8)
6. Удаление пересечений с тестовой выборкой (data leakage)

Результат сохраняется в `data/interim/cleaned.csv`.

### 01c_preprocessing.ipynb

Финальная обработка очищенного датасета:

1. Создание трёх версий текста: `text_raw`, `text_normalized`, `text_preprocessed`
2. Извлечение 20 числовых признаков (эмодзи, переводы строк, ссылки, длина, доля заглавных, пунктуация, цифры и др.)
3. Визуализация препроцессинга (до/после)
4. Удаление строк с пустым `text_preprocessed`
5. Анализ дисбаланса классов
6. Корреляция признаков с целевой переменной

Результат сохраняется в `data/processed/preprocessed.csv`.

### 01d_test_preparation.ipynb

Подготовка внешнего тестового датасета по аналогии с тренировочным пайплайном.

Источники тестовых данных:

- **test_clear.json** — чистые сообщения из открытых Telegram чатов различных ВУЗов (ham, label=0)
- **parsed_lols_2023.txt** — спам-сообщения с lols.bot, 2023 год (spam, label=1)

Результат сохраняется в `data/processed/test_preprocessed.csv` и используется в ноутбуках 05–07 для оценки моделей на внешних данных.
